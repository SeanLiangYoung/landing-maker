# Landing Maker 后续优化计划

本文档记录 Agent 体系的后续优化方向，供迭代参考。

---

## 1. DAG 知识库

为设计师和前端专家分别增加 DAG（有向无环图）知识库，用于结构化存储和检索专业知识。

| 知识库 | 使用方 | 内容方向 |
|--------|--------|----------|
| **设计师知识库** | 设计大师 Agent | 设计语言、版式规律、配色体系、行业案例、风格参考 |
| **前端专家知识库** | 前端专家 Agent | 实现模式、组件库、布局解法、动效实现、兼容性方案 |

**DAG 结构**：节点为知识点/模式，边为依赖关系（如「极简风格」依赖「留白」「无衬线字体」），支持按语义和依赖链检索。

---

## 2. 数据收集专家 Agent

新增 **数据收集专家** Agent，负责采集和精炼设计语言，持续丰富设计师知识库。

| 职责 | 说明 |
|------|------|
| **采集** | 搜集典型、经典的设计语言（网站、案例、设计系统、品牌规范等） |
| **精炼** | 对原始素材进行提炼、归纳、结构化处理 |
| **入库** | 将精炼后的内容存入设计师 DAG 知识库 |

**运行方式**：可定时触发或按需触发，与主流程（路径 1/2/3）解耦，作为后台知识维护流程。

---

## 3. 入口 Agent 能力增强

入口 Agent 增加对用户提交产品信息和文档的处理能力：

| 能力 | 说明 |
|------|------|
| **文档解析** | 支持用户提交产品介绍、PRD、品牌手册等文档 |
| **信息融合** | 将文档内容与对话中的补充信息相结合 |
| **需求总结** | 自动归纳、整理为结构化的 `demand.md` |
| **确认流程** | 将生成的 `demand.md` 呈交用户确认，根据反馈修正后再进入生成阶段 |

**输入形式**：支持文本粘贴、文件上传（如 PDF、Word、Markdown）、URL 引用等。

---

## 4. SEO 专家 Agent

新增 **SEO 专家** Agent，根据需求文档生成关键词与 SEO 建议，供前端专家进行搜索引擎优化。

| 职责 | 说明 |
|------|------|
| **输入** | 基于 `demand.md` 中的产品、内容、目标受众等信息 |
| **输出** | 关键词列表、meta 建议（title、description）、语义化标签建议等 |
| **传递** | 将 SEO 建议提供给前端专家 |

**前端专家使用**：在实现页面时，将 SEO 专家提供的关键词、meta 信息应用到 `<title>`、`<meta name="description">`、`<h1>` 层级、`alt` 等位置，提升页面在搜索引擎中的表现。

**流程位置**：在用户确认 `demand.md` 后、前端专家开始实现前，由 SEO 专家产出建议；或与设计大师并行执行（路径 1、3），与前端专家实现阶段合并输入。
