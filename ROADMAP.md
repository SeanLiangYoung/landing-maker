### 3. 评分系统 ✅

新增 **评分 Agent**，基于固定评分标准对 Landing 页面进行客观打分。

| 功能 | 说明 |
|------|------|
| **评分维度** | 需求符合度（25 分）、UI/UX 规范（30 分）、设计质量（20 分）、技术实现（10 分）、SEO 质量（15 分） |
| **合格标准** | 各大项 ≥80%，总分 ≥85 分 |
| **优化建议** | 为每个维度提供针对性的优化建议，按优先级（Critical/High/Medium）排序 |
| **迭代评估** | 支持迭代评估，对比改进情况 |

**实现位置**：`.cursor/skills/landing-scorer/SKILL.md`

| **头脑风暴总结** | 汇总各专家的建议，形成头脑风暴总结报告，交付给产品专家 |
| **任务编排** | 基于确认的 `demand.md`，协调设计大师、前端专家、SEO 专家等 Agent 执行任务 |
| **流程管理** | 管理整个生成流程的执行顺序和依赖关系 |
| **迭代协调** | 根据评分报告，协调相关 Agent 进行迭代优化 |
| **进度跟踪** | 跟踪各 Agent 的任务执行进度和状态 |
| **版本管理** | 管理页面版本（landing.html、landing-v2.html 等）和迭代历史 |
| **质量把关** | 确保评分合格后才交付，不合格时自动触发迭代 |
| **异常处理** | 处理迭代超限、Agent 执行失败等异常情况 |
| **项目复盘组织** | 迭代优化完成后，组织全体参与 Agent 进行项目复盘 |
| **复盘总结** | 对项目流程各个阶段进行复盘，识别各 Agent 的不足 |
| **能力提升建议** | 输出各个 Agent 能力提高的建议 |
| **技能优化** | 根据复盘结果，优化各个 Agent 的技能（Skill 文件） |

#### 职责边界

**项目经理负责**：
- ✅ 头脑风暴的组织和总结
- ✅ Agent 之间的协调和编排
- ✅ 任务分发和进度跟踪
- ✅ 迭代流程的管理
- ✅ 版本管理和迭代历史
- ✅ 质量把关和异常处理
- ✅ 项目复盘的组织和总结
- ✅ Agent 能力提升建议的输出
- ✅ Agent 技能的优化

**项目经理不负责**：
- ❌ 需求挖掘和内容整理（由产品专家负责）
- ❌ 具体的设计、开发、SEO 工作（由对应专家负责）

#### 与产品专家的协作

| 阶段 | 产品专家 | 项目经理 |
|------|----------|----------|
| **需求阶段** | 挖掘需求、生成初始 `demand.md` | 组织头脑风暴，总结建议 |
| **头脑风暴阶段** | 参与头脑风暴，提出产品角度建议 | 组织会议，汇总各专家建议，交付产品专家 |
| **需求确认阶段** | 根据头脑风暴结果更新 `demand.md`，向用户确认 | 等待需求确认 |
| **生成阶段** | 完成内容工作 | 协调各 Agent 执行任务 |
| **评分阶段** | - | 调用评分 Agent，检查合格性 |
| **迭代阶段** | 如需求问题，优化 `demand.md` | 协调相关 Agent 进行优化 |
| **复盘阶段** | 参与复盘，提供产品角度反馈 | 组织复盘，总结各 Agent 不足，输出能力提升建议，优化技能 |

#### 项目复盘功能

**复盘时机**：迭代优化阶段完成后（评分合格后）

**复盘流程**：
1. **组织复盘会议**：项目经理组织全体参与 Agent（产品专家、SEO 专家、设计大师、前端专家、评分 Agent）
2. **阶段复盘**：对项目流程各个阶段进行复盘
   - 需求挖掘阶段：产品专家的表现和不足
   - 头脑风暴阶段：各专家的参与度和建议质量
   - SEO 优化阶段：SEO 专家的输出质量和建议完整性
   - 生成阶段：设计大师和前端专家的实现质量
   - 评分阶段：评分 Agent 的评估准确性和建议有效性
   - 迭代优化阶段：各 Agent 的优化效果和响应速度
3. **问题识别**：明确各个 Agent 不足的地方
4. **能力提升建议**：输出各个 Agent 能力提高的建议
5. **技能优化**：根据复盘结果，优化各个 Agent 的技能（Skill 文件）
6. **复盘报告**：生成项目复盘报告，记录复盘结果和优化建议

**复盘输出**：
- `retrospective.md`：项目复盘报告，包含各阶段复盘结果、Agent 不足分析、能力提升建议
- Agent Skill 文件更新：根据复盘结果优化各 Agent 的技能定义

**预期效果**：
- **持续改进**：通过复盘持续优化各 Agent 的能力
- **问题发现**：及时发现和解决各 Agent 的不足
- **技能提升**：基于实际项目经验优化 Agent 技能

### 2. DAG 知识库

为产品专家、设计大师和前端专家分别建立 DAG（有向无环图）知识库，用于结构化存储和检索专业知识。

| 知识库 | 使用方 | 内容方向 | 侧重点 |
|--------|--------|----------|--------|
| **产品知识库** | 产品专家 Agent | 行业解决方案、标准文字内容、需求模板、产品定位模式 | **侧重积累行业的解决方案和标准的文字内容** |
| **设计师知识库** | 设计大师 Agent | 设计模板、设计语言、版式规律、配色体系、行业案例、风格参考 | **侧重积累很多套不同的设计模板** |
| **前端专家知识库** | 前端专家 Agent | 前端页面范式、展示模式、实现模式、组件库、布局解法、动效实现、兼容性方案 | **侧重积累前端页面和展示模式的标准范式** |

#### 产品知识库

**核心内容**：
- **行业解决方案**：不同行业（SaaS、电商、教育、医疗等）的 Landing 页面解决方案
- **标准文字内容**：行业通用的文案模板、价值主张表达、功能描述、定价说明等
- **需求模板**：常见产品类型的需求挖掘模板和问题清单
- **产品定位模式**：不同产品定位的表达方式和内容结构

**应用场景**：
- 产品专家在需求挖掘时，参考行业解决方案快速理解用户需求
- 生成 `demand.md` 时，使用标准文字内容模板，提升内容质量
- 头脑风暴时，提供行业最佳实践参考

**预期效果**：
- 提升产品专家的需求挖掘效率和质量
- 确保生成的内容符合行业标准
- 减少重复工作，提升内容一致性

#### 设计师知识库

**核心内容**：
- **设计模板库**：多套不同风格、不同行业的 Landing 页面设计模板
  - 极简风格模板
  - 商务专业模板
  - 创意活力模板
  - 科技感模板
  - 等各类设计模板
- **设计语言**：设计元素、视觉风格、设计原则
- **版式规律**：布局模式、信息架构、视觉流
- **配色体系**：配色方案、色彩心理学应用
- **行业案例**：不同行业的优秀设计案例
- **风格参考**：各种设计风格的参考和说明

**应用场景**：
- 设计大师在设计时，参考设计模板快速产出设计规格
- 路径 3（参考+创意）时，快速找到相似风格的设计模板
- 头脑风暴时，提供设计角度的建议和参考

**预期效果**：
- 提升设计大师的设计效率和质量
- 提供丰富的设计模板选择，满足不同需求
- 确保设计风格的一致性和专业性

#### 前端专家知识库

**核心内容**：
- **前端页面范式**：标准化的页面结构和组织方式
- **展示模式**：常见的内容展示模式（Hero、Features、Testimonials、Pricing、FAQ、CTA 等）
- **实现模式**：代码实现的最佳实践和模式
- **组件库**：可复用的前端组件和代码片段
- **布局解法**：响应式布局、Grid/Flexbox 布局方案
- **动效实现**：常见动效的实现方案和代码
- **兼容性方案**：浏览器兼容性处理和解决方案

**应用场景**：
- 前端专家在实现时，参考标准范式快速构建页面
- 路径 2（内容工具）时，选择合适的展示模式组织内容
- 头脑风暴时，提供技术可行性建议

**预期效果**：
- 提升前端专家的实现效率和质量
- 确保代码质量和一致性
- 减少重复开发，提升可维护性

#### DAG 结构

**节点**：知识点/模式（如「极简风格」「Hero 区块」「响应式布局」）
**边**：依赖关系（如「极简风格」依赖「留白」「无衬线字体」）
**检索**：支持按语义和依赖链检索

**知识库关系**：
- 产品知识库 → 设计师知识库：产品定位影响设计风格选择
- 设计师知识库 → 前端专家知识库：设计规格影响实现方式
- 三个知识库相互关联，形成完整的知识体系

**预期效果**：
- 提升各 Agent 的专业能力和效率
- 支持知识积累和复用
- 形成持续改进的知识体系

**预期效果**：
