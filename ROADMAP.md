# Landing Maker 后续优化计划

本文档记录 Agent 体系的后续优化方向，供迭代参考。

---

## ✅ 已完成功能

### 1. 产品专家 Agent ✅

**产品专家**专注于内容相关的工作，负责需求挖掘和内容整理。

#### 核心职责（内容相关）

| 职责 | 说明 | 状态 |
|------|------|------|
| **文档解析** | 支持用户提交产品介绍、PRD、品牌手册等文档，提取关键信息 | ✅ |
| **信息融合** | 将文档内容与对话中的补充信息相结合，形成完整的需求画像 | ✅ |
| **多轮对话** | 通过专业的提问技巧，渐进式提问，逐步明确客户需求 | ✅ |
| **需求挖掘** | 深度挖掘产品定位、目标用户、核心功能、品牌规范等内容 | ✅ |
| **内容整理** | 将收集到的信息整理为结构化的 `demand.md` | ✅ |
| **路径选择引导** | 基于需求特点，引导用户选择最适合的设计路径 | ✅ |
| **需求确认** | 将生成的 `demand.md` 呈交用户确认，根据反馈修正 | ✅ |
| **内容优化** | 根据评分报告中的需求符合度问题，优化需求内容 | ✅ |

#### 职责边界

**产品专家负责**：
- ✅ 产品文档解析和信息提取
- ✅ 多轮对话挖掘需求
- ✅ 需求内容整理和结构化
- ✅ 生成和更新 `demand.md`
- ✅ 需求确认和内容修正

**产品专家不负责**：
- ❌ Agent 之间的协调和编排
- ❌ 迭代流程的管理
- ❌ 任务分发和进度跟踪

**实现位置**：`.cursor/skills/landing-orchestrator/SKILL.md`（未来将拆分为 `landing-product-expert/SKILL.md`）

### 2. SEO 专家 Agent ✅

新增 **SEO 专家** Agent，根据需求文档生成关键词与 SEO 建议，供前端专家进行搜索引擎优化。

| 职责 | 说明 |
|------|------|
| **输入** | 基于 `demand.md` 中的产品、内容、目标受众等信息 |
| **输出** | 关键词策略、meta 标签建议、标题结构建议、结构化数据建议、AI-Optimized SEO 建议 |
| **迭代优化** | 根据评分报告优化 `seo-suggestions.md` |

**前端专家使用**：在实现页面时，完整应用 SEO 专家提供的所有建议，包括 meta 标签、标题结构、关键词、结构化数据 JSON-LD 等。

**实现位置**：`.cursor/skills/landing-seo-expert/SKILL.md`

---

### 3. 评分系统 ✅

新增 **评分 Agent**，基于固定评分标准对 Landing 页面进行客观打分。

| 功能 | 说明 |
|------|------|
| **评分维度** | 需求符合度（25 分）、UI/UX 规范（30 分）、设计质量（20 分）、技术实现（10 分）、SEO 质量（15 分） |
| **合格标准** | 各大项 ≥80%，总分 ≥85 分 |
| **优化建议** | 为每个维度提供针对性的优化建议，按优先级（Critical/High/Medium）排序 |
| **迭代评估** | 支持迭代评估，对比改进情况 |

**实现位置**：`.cursor/skills/landing-scorer/SKILL.md`

---

### 4. 迭代优化流程 ✅

实现基于评分报告的自动迭代优化流程。

| 功能 | 说明 |
|------|------|
| **自动触发** | 评分不合格时自动触发迭代优化 |
| **针对性优化** | 根据未达标维度确定需要优化的 Agent |
| **协调机制** | 项目经理协调各 Agent 进行优化（当前由产品专家临时承担） |
| **版本管理** | 保留迭代历史（landing.html、landing-v2.html 等） |
| **迭代限制** | 最多迭代 5 次，避免无限循环 |

**实现位置**：
- `.cursor/rules/landing-maker-workflow.mdc`（工作流规则）
- `.cursor/skills/landing-frontend-expert/SKILL.md`（前端专家迭代流程）
- `.cursor/skills/landing-design-master/SKILL.md`（设计大师迭代流程）
- `.cursor/skills/landing-seo-expert/SKILL.md`（SEO 专家迭代流程）

**注意**：当前迭代管理功能由产品专家临时承担，未来将移交给项目经理 Agent。

## 🚧 规划中功能

### 1. 项目经理 Agent 🚧 **高优先级**

新增 **项目经理 Agent**，专门负责 Agent 协调、任务编排和迭代流程管理。

#### 核心职责（协调管理相关）

| 职责 | 说明 |
|------|------|
| **头脑风暴组织** | 需求挖掘完成后，组织产品专家、SEO 专家、设计大师、前端专家进行头脑风暴，针对需求文档提出多角度建议 |
| **头脑风暴总结** | 汇总各专家的建议，形成头脑风暴总结报告，交付给产品专家 |
| **任务编排** | 基于确认的 `demand.md`，协调设计大师、前端专家、SEO 专家等 Agent 执行任务 |
| **流程管理** | 管理整个生成流程的执行顺序和依赖关系 |
| **迭代协调** | 根据评分报告，协调相关 Agent 进行迭代优化 |
| **进度跟踪** | 跟踪各 Agent 的任务执行进度和状态 |
| **版本管理** | 管理页面版本（landing.html、landing-v2.html 等）和迭代历史 |
| **质量把关** | 确保评分合格后才交付，不合格时自动触发迭代 |
| **异常处理** | 处理迭代超限、Agent 执行失败等异常情况 |
| **项目复盘组织** | 迭代优化完成后，组织全体参与 Agent 进行项目复盘 |
| **复盘总结** | 对项目流程各个阶段进行复盘，识别各 Agent 的不足 |
| **能力提升建议** | 输出各个 Agent 能力提高的建议 |
| **技能优化** | 根据复盘结果，优化各个 Agent 的技能（Skill 文件） |

#### 职责边界

**项目经理负责**：
- ✅ 头脑风暴的组织和总结
- ✅ Agent 之间的协调和编排
- ✅ 任务分发和进度跟踪
- ✅ 迭代流程的管理
- ✅ 版本管理和迭代历史
- ✅ 质量把关和异常处理
- ✅ 项目复盘的组织和总结
- ✅ Agent 能力提升建议的输出
- ✅ Agent 技能的优化

**项目经理不负责**：
- ❌ 需求挖掘和内容整理（由产品专家负责）
- ❌ 具体的设计、开发、SEO 工作（由对应专家负责）

#### 与产品专家的协作

| 阶段 | 产品专家 | 项目经理 |
|------|----------|----------|
| **需求阶段** | 挖掘需求、生成初始 `demand.md` | 组织头脑风暴，总结建议 |
| **头脑风暴阶段** | 参与头脑风暴，提出产品角度建议 | 组织会议，汇总各专家建议，交付产品专家 |
| **需求确认阶段** | 根据头脑风暴结果更新 `demand.md`，向用户确认 | 等待需求确认 |
| **生成阶段** | 完成内容工作 | 协调各 Agent 执行任务 |
| **评分阶段** | - | 调用评分 Agent，检查合格性 |
| **迭代阶段** | 如需求问题，优化 `demand.md` | 协调相关 Agent 进行优化 |
| **复盘阶段** | 参与复盘，提供产品角度反馈 | 组织复盘，总结各 Agent 不足，输出能力提升建议，优化技能 |

#### 项目复盘功能

**复盘时机**：迭代优化阶段完成后（评分合格后）

**复盘流程**：
1. **组织复盘会议**：项目经理组织全体参与 Agent（产品专家、SEO 专家、设计大师、前端专家、评分 Agent）
2. **阶段复盘**：对项目流程各个阶段进行复盘
   - 需求挖掘阶段：产品专家的表现和不足
   - 头脑风暴阶段：各专家的参与度和建议质量
   - SEO 优化阶段：SEO 专家的输出质量和建议完整性
   - 生成阶段：设计大师和前端专家的实现质量
   - 评分阶段：评分 Agent 的评估准确性和建议有效性
   - 迭代优化阶段：各 Agent 的优化效果和响应速度
3. **问题识别**：明确各个 Agent 不足的地方
4. **能力提升建议**：输出各个 Agent 能力提高的建议
5. **技能优化**：根据复盘结果，优化各个 Agent 的技能（Skill 文件）
6. **复盘报告**：生成项目复盘报告，记录复盘结果和优化建议

**复盘输出**：
- `retrospective.md`：项目复盘报告，包含各阶段复盘结果、Agent 不足分析、能力提升建议
- Agent Skill 文件更新：根据复盘结果优化各 Agent 的技能定义

**预期效果**：
- **持续改进**：通过复盘持续优化各 Agent 的能力
- **问题发现**：及时发现和解决各 Agent 的不足
- **技能提升**：基于实际项目经验优化 Agent 技能
- **知识积累**：形成项目经验库，指导后续项目

#### 预期效果

- **职责清晰**：产品专家专注内容，项目经理专注协调
- **流程规范**：统一的流程管理和任务编排
- **质量保证**：自动化的质量把关和迭代管理
- **可维护性**：清晰的职责边界，便于维护和扩展
- **持续改进**：通过项目复盘持续优化 Agent 能力

**实现位置**：`.cursor/skills/landing-project-manager/SKILL.md`（待创建）

#### 职责划分总结

| Agent | 核心职责 | 工作内容 |
|-------|----------|----------|
| **产品专家** | 内容挖掘和需求整理 | 文档解析、多轮对话、需求挖掘、内容整理、生成 `demand.md`、根据头脑风暴结果更新需求 |
| **项目经理** | 协调管理和流程控制 | 头脑风暴组织、任务编排、Agent 协调、流程管理、迭代协调、质量把关 |
| **设计大师** | 设计规格产出 | 路径 1、3 下产出设计规格 |
| **前端专家** | 页面实现 | 实现设计规格或直接实现（路径 2） |
| **SEO 专家** | SEO 建议产出 | 生成 SEO 优化建议 |
| **评分 Agent** | 质量评估 | 评分、合格性检查、优化建议 |

**关键原则**：
- 产品专家专注**内容**，不负责协调管理
- 项目经理专注**协调**，不负责内容挖掘
- 职责边界清晰，便于维护和扩展

### 2. DAG 知识库

为设计师和前端专家分别增加 DAG（有向无环图）知识库，用于结构化存储和检索专业知识。

| 知识库 | 使用方 | 内容方向 |
|--------|--------|----------|
| **设计师知识库** | 设计大师 Agent | 设计语言、版式规律、配色体系、行业案例、风格参考 |
| **前端专家知识库** | 前端专家 Agent | 实现模式、组件库、布局解法、动效实现、兼容性方案 |

**DAG 结构**：节点为知识点/模式，边为依赖关系（如「极简风格」依赖「留白」「无衬线字体」），支持按语义和依赖链检索。

**预期效果**：
- 提升设计大师的设计质量和创意度
- 提升前端专家的实现效率和质量
- 支持知识积累和复用

### 3. 数据收集专家 Agent

新增 **数据收集专家** Agent，负责采集和精炼设计语言，持续丰富设计师知识库。

| 职责 | 说明 |
|------|------|
| **采集** | 搜集典型、经典的设计语言（网站、案例、设计系统、品牌规范等） |
| **精炼** | 对原始素材进行提炼、归纳、结构化处理 |
| **入库** | 将精炼后的内容存入设计师 DAG 知识库 |

**运行方式**：可定时触发或按需触发，与主流程（路径 1/2/3）解耦，作为后台知识维护流程。

**预期效果**：
- 持续丰富设计师知识库
- 提升设计大师的设计参考能力
- 支持设计趋势跟踪
