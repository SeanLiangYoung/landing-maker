# 多模态内容增强方案

**创建时间**：2026-02-10  
**目标**：引入多模态模型（图片/视频生成），丰富 Landing 页面内容

---

## 一、问题分析

### 1.1 当前问题

1. **视觉内容缺失**：
   - 只有 SVG 图标，缺少真实的产品截图、功能演示图
   - 没有产品演示视频、用户故事视频
   - Hero 区块只有文字和渐变背景，缺少视觉吸引力

2. **内容单薄**：
   - 案例展示只有文字描述，缺少视觉对比
   - 功能展示只有图标和文字，缺少实际效果展示
   - 数据展示只有数字，缺少可视化图表

3. **用户体验不足**：
   - 缺少产品实际使用场景的视觉呈现
   - 缺少产品效果的直观展示（如：简历优化前后对比图）
   - 缺少动态内容（视频）吸引用户注意力

---

## 二、解决方案架构

### 2.1 新增 Agent：多模态内容生成专家

**Agent 名称**：`landing-multimodal-expert`

**职责**：
- 根据 `demand.md` 和 `design-spec.md` 规划需要生成的图片和视频
- 调用多模态模型 API 生成图片和视频
- 管理生成的媒体资源（存储、优化、引用）
- 为前端专家提供媒体资源路径和引用方式

**输入**：
- `demand.md`：产品需求文档
- `design-spec.md`：设计规格文档
- 设计路径（路径 1/2/3）

**输出**：
- `multimodal-assets.md`：媒体资源清单和生成说明
- `assets/` 目录：生成的图片和视频文件
- 媒体资源引用指南

---

### 2.2 多模态内容类型

#### 2.2.1 图片类型

1. **Hero 区块图片**：
   - 产品主图：展示产品核心功能的视觉化图片
   - 产品截图：产品界面截图（如：简历优化工具界面）
   - 背景装饰图：增强视觉吸引力的装饰性图片

2. **功能展示图片**：
   - 功能演示图：每个功能的实际使用效果图
   - 对比图：优化前后的对比图（如：简历优化前后对比）
   - 流程图：功能使用流程的可视化图表

3. **案例展示图片**：
   - 用户头像：用户头像图片（可生成或使用占位符）
   - 案例对比图：案例中的前后对比图
   - 数据可视化图：案例中的数据可视化图表

4. **数据展示图片**：
   - 数据图表：用户数量、优化简历数量等数据的可视化图表
   - 统计图：通过率提升、满意度等统计图表

5. **定价展示图片**：
   - 套餐对比图：不同套餐的对比图表
   - 价值展示图：套餐价值的可视化展示

#### 2.2.2 视频类型

1. **产品演示视频**：
   - 产品功能演示：展示产品核心功能的使用过程
   - 产品介绍视频：产品整体介绍视频（30-60 秒）

2. **用户故事视频**：
   - 用户案例视频：真实用户使用产品的故事视频
   - 成功案例视频：成功案例的短视频（15-30 秒）

3. **功能演示视频**：
   - 功能使用视频：每个功能的详细使用演示
   - 对比视频：优化前后的对比视频

### 2.3 API 集成方案

#### 2.3.1 图片生成 API

**选项 1：DALL-E 3 / Midjourney API**
- **优势**：生成质量高，风格多样
- **适用场景**：产品主图、功能演示图、装饰性图片
- **API 调用**：OpenAI DALL-E 3 API 或 Midjourney API

**选项 2：Stable Diffusion API**
- **优势**：开源，可定制，成本较低
- **适用场景**：产品截图、功能演示图
- **API 调用**：Stable Diffusion API（如：Replicate、Hugging Face）

**选项 3：专业设计工具 API**
- **优势**：专业设计，符合品牌规范
- **适用场景**：产品主图、品牌图片
- **API 调用**：Canva API、Figma API 等

#### 2.3.2 视频生成 API

**选项 1：Runway ML / Pika Labs API**
- **优势**：AI 视频生成，质量高
- **适用场景**：产品演示视频、功能演示视频
- **API 调用**：Runway ML API 或 Pika Labs API

**选项 2：Synthesia / D-ID API**
- **优势**：AI 虚拟人物视频，专业
- **适用场景**：产品介绍视频、用户故事视频
- **API 调用**：Synthesia API 或 D-ID API

**选项 3：图片转视频 API**
- **优势**：基于图片生成视频，成本较低
- **适用场景**：产品演示视频、功能演示视频
- **API 调用**：LeiaPix API、Stable Video Diffusion API

## 三、工作流程设计

### 3.1 集成到现有流程

```
1. 产品专家 → 生成 demand.md
2. 项目经理 → 组织头脑风暴
3. 产品专家 → 更新 demand.md（v1.1）
4. 用户确认 → demand.md（v1.2）
5. SEO 专家 → 生成 seo-suggestions.md
6. 设计大师 → 生成 design-spec.md
7. **多模态内容生成专家 → 生成图片和视频**（新增）
   - 根据 demand.md 和 design-spec.md 规划媒体资源
   - 调用 API 生成图片和视频
   - 生成 multimodal-assets.md
8. 前端专家 → 实现页面（参考设计规格和媒体资源）
9. 评分 Agent → 评分
10. 项目经理 → 项目复盘
```

### 3.2 多模态内容生成专家工作流程

1. **内容规划阶段**：
   - 读取 `demand.md` 和 `design-spec.md`
   - 分析需要生成的图片和视频类型

2. **图片生成阶段**：
   - 根据规划清单，调用图片生成 API
   - 生成各类图片（Hero 图片、功能演示图、案例对比图等）
   - 优化图片（压缩、格式转换、尺寸调整）

3. **视频生成阶段**：
   - 根据规划清单，调用视频生成 API
   - 生成各类视频（产品演示视频、用户故事视频等）
   - 优化视频（压缩、格式转换、时长调整）

4. **资源管理阶段**：
   - 将生成的媒体资源保存到 `assets/` 目录
   - 生成 `multimodal-assets.md` 资源清单
   - 为前端专家提供资源引用指南

## 七、技术挑战与解决方案

### 7.1 API 成本控制

**挑战**：图片和视频生成 API 调用成本较高

**解决方案**：
- 使用缓存机制，避免重复生成
- 使用成本较低的 API 作为备选（如：Stable Diffusion）
- 批量生成，提高效率
- 使用 CDN 存储，减少重复下载

### 7.2 生成内容质量

**挑战**：AI 生成的内容质量不稳定

**解决方案**：
- 优化提示词（Prompt Engineering）
- 使用质量检查机制
- 提供人工审核选项
- 使用多个 API 备选方案

### 7.3 生成时间控制

**挑战**：图片和视频生成需要时间

**解决方案**：
- 异步生成，不阻塞主流程
- 使用占位符，先展示页面结构
- 批量生成，提高效率
- 使用缓存，避免重复生成

### 7.4 存储管理

**挑战**：生成的媒体资源需要存储和管理

**解决方案**：
- 本地存储 + 云存储（Cloudinary、AWS S3）
- 自动压缩和优化
- 版本管理（避免重复生成）
- 清理机制（删除未使用的资源）

## 九、前端专家集成

### 9.1 媒体资源引用

前端专家在实现页面时，需要：

1. **读取媒体资源清单**：
   - 读取 `multimodal-assets.md`
   - 了解可用的图片和视频资源

2. **引用媒体资源**：
   - 在 HTML 中引用图片：`<img src="assets/images/hero/hero-main.png" alt="...">`
   - 在 HTML 中引用视频：`<video src="assets/videos/product/product-demo.mp4"></video>`

3. **优化媒体资源**：
   - 使用响应式图片（`srcset`、`sizes`）
   - 使用懒加载（`loading="lazy"`）
   - 使用 WebP 格式（现代浏览器）

### 9.2 设计规格更新

设计规格需要包含：

1. **媒体资源要求**：
   - 每个区块需要的图片类型和尺寸
   - 每个区块需要的视频类型和时长

2. **媒体资源规范**：
   - 图片格式（PNG、JPEG、WebP）
   - 图片尺寸（桌面端、移动端）
   - 视频格式（MP4、WebM）
   - 视频时长和分辨率
