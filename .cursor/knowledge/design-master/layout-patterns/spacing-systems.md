# 间距系统知识

## 留白（Generous Whitespace）

### 定义
充足的留白空间，创造呼吸感和层次感。

### 应用原则
- **区块间距**：80-120px（大区块之间）
- **卡片内边距**：32-48px（卡片内部内容与边缘的距离）
- **元素间距**：16-24px（相关元素之间）
- **文字行距**：1.6-1.8（正文行高）

### 参考案例
- Apple 官网：大量留白，创造高端感
- Stripe 官网：留白充足，信息清晰

### 设计建议
- 避免内容过于拥挤
- 留白即设计，不要害怕留白
- 根据内容重要性调整留白大小

## 一致间距（Consistent Spacing）

### 定义
使用一致的间距系统，保持视觉统一。

### 间距系统
- **基础单位**：8px 或 16px
- **间距倍数**：8px, 16px, 24px, 32px, 40px, 48px, 64px, 80px, 96px, 120px
- **应用规则**：使用基础单位的倍数，保持一致性

### 应用场景
- 网格系统间距
- 元素之间的间距
- 区块之间的间距

### 设计建议
- 选择 8px 或 16px 作为基础单位
- 所有间距都应该是基础单位的倍数
- 建立间距系统文档，确保一致性

---

## 网格系统（Grid System）

### 定义
使用网格系统组织内容，保持对齐和一致性。

### 常见网格
- **12列网格**：适合复杂布局
- **CSS Grid**：灵活的网格系统
- **Flexbox**：一维布局系统

### 应用建议
- **容器最大宽度**：1200px（桌面端）
- **列间距（Gutter）**：24-32px
- **响应式断点**：375px, 768px, 1024px, 1440px

### 设计建议
- 使用 CSS Grid 或 Flexbox 实现
- 保持列间距一致
- 响应式设计，在不同断点下调整网格

## 布局类型（Layout Types）

### 居中Hero（Hero Centered）
- **特点**：Hero 区块内容居中
- **应用**：适合简洁、专业的页面
- **留白**：上下留白充足（80-120px）

### 分屏Hero（Hero Split）
- **特点**：左右分屏，左侧文字，右侧图片/视频
- **应用**：适合需要展示产品的页面
- **对齐**：使用网格系统对齐

### 卡片布局（Card Layout）
- **特点**：使用卡片组织内容
- **应用**：功能展示、定价方案、案例展示
- **间距**：卡片之间间距 32px，卡片内边距 32-40px

### 功能网格（Feature Grid）
- **特点**：使用网格展示功能特性
- **应用**：核心功能展示
- **布局**：根据功能数量选择 2×2、3列等布局
