# 设计大师知识库

设计大师 Agent 的 DAG（有向无环图）知识库，用于结构化存储和检索专业知识。

## 知识库结构

### DAG 结构说明

- **节点**：知识点/模式（如设计风格、版式规律、配色方案等）
- **边**：依赖关系（如「极简风格」依赖「留白」「无衬线字体」）
- **检索方式**：支持按语义和依赖链检索

### 知识分类

1. **设计语言**：设计风格、视觉语言、设计原则
2. **版式规律**：布局模式、间距系统、网格系统
3. **配色体系**：配色方案、色彩理论、品牌色应用
4. **行业案例**：不同行业的优秀设计案例
5. **风格参考**：经典设计风格、现代设计趋势

## 文件结构

```

├── CHANGELOG.md                 # 知识库变更日志
├── knowledge-dag.json          # DAG 结构定义（节点和边）
├── design-languages/           # 设计语言知识
│   ├── styles/                # 设计风格
│   ├── principles/            # 设计原则
│   └── visual-languages/      # 视觉语言
├── layout-patterns/            # 版式规律知识
│   ├── grid-systems/          # 网格系统
│   ├── spacing-systems/       # 间距系统
│   └── layout-types/         # 布局类型
├── color-systems/             # 配色体系知识
│   ├── palettes/              # 配色方案
│   ├── color-theory/          # 色彩理论
│   └── brand-colors/          # 品牌色应用
├── industry-cases/            # 行业案例知识
│   ├── saas/                  # SaaS 行业案例
│   ├── e-commerce/            # 电商行业案例
│   ├── fintech/               # 金融科技案例

    ├── classic/               # 经典风格

    └── trends/                # 设计趋势
```

## 使用方法

设计大师 Agent 在设计时，可以：
1. 根据需求检索相关的设计知识
2. 根据依赖关系查找相关的设计模式
3. 参考行业案例和风格参考
4. 应用版式规律和配色体系

## 知识库维护

- **数据收集专家 Agent**：负责采集和精炼设计语言，持续丰富知识库
- **项目复盘**：根据实际项目经验，持续优化知识库内容
- **变更日志**：所有知识库更新都会记录在 `CHANGELOG.md` 中，包括新增、更新、删除的知识点和 DAG 变更
