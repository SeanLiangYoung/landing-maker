# SaaS 产品需求挖掘模板

## 需求挖掘流程

### 阶段 1：产品基础信息

#### 1.1 产品定位
- **问题**：您的产品是什么？主要解决什么问题？
- **目标**：了解产品核心定位和价值
- **示例**：CRM 系统、项目管理工具、数据分析平台

#### 1.2 目标用户
- **问题**：主要面向哪些用户？（企业规模、行业、角色）
- **目标**：明确目标用户画像
- **示例**：中小企业、特定行业、特定角色（如销售、市场）

#### 1.3 核心功能
- **问题**：产品的核心功能有哪些？（3-5 个）
- **目标**：了解产品功能范围
- **示例**：任务管理、团队协作、数据分析、自动化

### 阶段 2：商业模式

#### 2.1 定价模式
- **问题**：采用什么定价模式？（免费版、订阅制、按量付费）
- **目标**：了解商业模式
- **示例**：免费版 + 专业版（$X/月）+ 企业版（定制价格）

#### 2.2 定价层级
- **问题**：有哪些定价层级？每个层级的价格和功能是什么？
- **目标**：了解定价结构
- **示例**：免费版（核心功能）、专业版（$29/月，全部功能）、企业版（定制）

#### 2.3 免费试用
- **问题**：是否提供免费试用？试用期多长？
- **目标**：了解试用机制
- **示例**：14 天免费试用、免费版永久使用

### 阶段 3：竞争优势

#### 3.1 核心优势
- **问题**：与竞品相比，主要优势是什么？
- **目标**：了解差异化优势
- **示例**：更快的速度、更好的用户体验、更强的功能

#### 3.2 独特卖点
- **问题**：有哪些独特的卖点？
- **目标**：了解产品亮点
- **示例**：AI 智能推荐、无缝集成、数据可视化

#### 3.3 选择理由
- **问题**：客户选择我们的主要原因是什么？
- **目标**：了解客户决策因素
- **示例**：性价比高、功能强大、服务完善

### 阶段 4：社会证明

#### 4.1 客户案例
- **问题**：有哪些知名客户或案例？
- **目标**：收集社会证明素材
- **示例**：知名企业客户、成功案例、使用场景

#### 4.2 用户评价
- **问题**：有哪些用户评价或反馈？
- **目标**：收集用户评价
- **示例**：用户评价、使用反馈、满意度数据

#### 4.3 统计数据
- **问题**：有哪些统计数据可以展示？
- **目标**：收集数据支撑
- **示例**：用户数量、使用时长、满意度、效率提升数据

### 阶段 5：技术集成

#### 5.1 集成需求
- **问题**：需要与哪些工具集成？
- **目标**：了解集成需求
- **示例**：Slack、Google Workspace、Salesforce

#### 5.2 API 支持
- **问题**：是否提供 API？面向开发者吗？
- **目标**：了解技术能力
- **示例**：REST API、Webhook、SDK

### 阶段 6：设计偏好

#### 6.1 风格偏好
- **问题**：希望页面呈现什么风格？（极简、专业、现代、创意）
- **目标**：了解设计偏好
- **示例**：极简专业、现代科技、温暖友好

#### 6.2 品牌规范
- **问题**：是否有品牌色、字体等规范？
- **目标**：了解品牌要求
- **示例**：品牌色 #007bff、使用系统字体

#### 6.3 禁止元素
- **问题**：是否有禁止使用的元素或风格？
- **目标**：了解设计约束
- **示例**：不使用紫色、不使用 emoji

## 需求挖掘技巧

### 1. 渐进式提问
- 不急于一次问完所有问题
- 每次聚焦 1-2 个维度
- 根据回答逐步深入

### 2. 引导式提问
- 基于初步信息引导用户
- 提供选项帮助用户选择
- 使用示例帮助用户理解

### 3. 确认式提问
- 总结用户回答
- 确认理解是否正确
- 补充遗漏信息
