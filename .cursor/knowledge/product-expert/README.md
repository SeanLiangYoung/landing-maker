# 产品专家知识库

产品专家 Agent 的 DAG（有向无环图）知识库，用于结构化存储和检索产品专业知识。

## 知识库结构

### DAG 结构说明

- **节点**：知识点/模式（如行业解决方案、标准文字内容、需求模板、产品定位模式等）
- **边**：依赖关系（如「SaaS 产品」依赖「B2B 定位」「功能展示模式」）
- **检索方式**：支持按语义和依赖链检索

### 知识分类

1. **行业解决方案**：不同行业的 Landing 页面解决方案（SaaS、电商、教育、医疗等）
2. **标准文字内容**：行业通用的文案模板、价值主张表达、功能描述、定价说明等
3. **需求模板**：常见产品类型的需求挖掘模板和问题清单
4. **产品定位模式**：不同产品定位的表达方式和内容结构

## 文件结构

```
product-expert/
├── README.md                    # 知识库说明文档
├── CHANGELOG.md                 # 知识库变更日志
├── knowledge-dag.json          # DAG 结构定义（节点和边）
├── industry-solutions/         # 行业解决方案知识
│   ├── saas/                  # SaaS 行业解决方案
│   ├── e-commerce/            # 电商行业解决方案
│   ├── education/             # 教育行业解决方案
│   ├── healthcare/            # 医疗行业解决方案
│   └── fintech/               # 金融科技行业解决方案
├── standard-content/          # 标准文字内容知识
│   ├── value-propositions/    # 价值主张表达
│   ├── feature-descriptions/  # 功能描述模板
│   ├── pricing-copy/          # 定价说明模板
│   └── cta-copy/              # CTA 文案模板
├── requirement-templates/     # 需求模板知识
│   ├── saas-template/         # SaaS 产品需求模板
│   ├── e-commerce-template/   # 电商产品需求模板
│   └── common-template/       # 通用需求模板
└── positioning-patterns/      # 产品定位模式知识
    ├── b2b/                   # B2B 定位模式
    ├── b2c/                   # B2C 定位模式
    └── hybrid/                # 混合定位模式
```

## 使用方法

产品专家 Agent 在需求挖掘时，可以：
1. 根据产品行业检索相关的行业解决方案
2. 根据依赖关系查找相关的标准内容模板
3. 参考需求模板，使用标准的问题清单
4. 应用产品定位模式，优化内容表达

## 知识库维护

- **数据收集专家 Agent**：负责采集和精炼产品知识，持续丰富知识库
- **项目复盘**：根据实际项目经验，持续优化知识库内容
- **最佳实践积累**：将优秀的行业解决方案和标准内容纳入知识库
- **变更日志**：所有知识库更新都会记录在 `CHANGELOG.md` 中，包括新增、更新、删除的知识点和 DAG 变更
