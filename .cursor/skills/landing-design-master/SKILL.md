# Landing 设计大师（Design Master）

负责将需求转化为具有专业美感的设计规格，输出可被前端精确实现的设计文档。

**设计时需参考并遵循**：
- [frontend-design](.cursor/skills/frontend-design/SKILL.md) 的美学原则
- [ui-ux-pro-max](.cursor/skills/ui-ux-pro-max/SKILL.md) 的行业规范与可访问性要求
- **设计大师知识库**：`.cursor/knowledge/design-master/` - DAG 知识库，包含设计语言、版式规律、配色体系、行业案例、风格参考

## 知识库使用

设计大师拥有专门的 DAG（有向无环图）知识库，用于结构化存储和检索专业知识。

### 知识库位置
`.cursor/knowledge/design-master/`

### 知识库结构
- **knowledge-dag.json**：DAG 结构定义（节点和边），包含知识点之间的依赖关系
- **design-languages/**：设计语言知识（设计风格、设计原则、视觉语言）
- **layout-patterns/**：版式规律知识（网格系统、间距系统、布局类型）
- **color-systems/**：配色体系知识（配色方案、色彩理论、品牌色应用）
- **industry-cases/**：行业案例知识（SaaS、电商、金融科技等）
- **style-references/**：风格参考知识（经典风格、现代风格、设计趋势）

### 知识库使用方法

1. **检索相关设计知识**：
   - 根据需求中的风格、情绪基调、行业等信息，检索相关的设计知识
   - 例如：需求中提到"极简风格"，检索 `minimal-style` 节点及其依赖知识

2. **根据依赖关系查找设计模式**：
   - 使用 DAG 的依赖关系，查找相关的设计模式
   - 例如：选择"极简风格"时，自动查找其依赖的"留白"、"无衬线字体"、"有限配色"等知识

3. **参考行业案例和风格参考**：
   - 根据产品行业，参考对应的行业案例
   - 根据设计趋势，参考现代设计风格

4. **应用版式规律和配色体系**：
   - 根据需求选择合适的版式规律（网格系统、间距系统、布局类型）
   - 根据需求选择合适的配色体系（有限配色、温暖配色、保守配色等）

### 知识库检索示例

**场景 1**：用户需求"极简风格、专业可信"
- 检索 `minimal-style` 和 `professional-trustworthy` 节点
- 查找依赖关系：`generous-whitespace`、`sans-serif-fonts`、`structured-layout` 等
- 参考 SaaS 行业案例
- 应用相应的版式规律和配色体系

**场景 2**：用户需求"温暖友好、面向消费者"
- 检索 `warm-friendly` 节点
- 查找依赖关系：`warm-color-palette`、`rounded-corners`、`soft-shadows` 等
- 参考相关行业案例
- 应用相应的视觉语言和配色方案

### 知识库维护

- **数据收集专家 Agent**：负责采集和精炼设计语言，持续丰富知识库
- **项目复盘**：根据实际项目经验，持续优化知识库内容

## 设计路径

本 Skill 在 **路径 1（纯创意）** 和 **路径 3（参考+创意）** 下主导设计；**路径 2（内容工具）** 由前端专家主导，但设计大师应积极参与，提供具体的设计建议。

### 路径 1：纯创意

- **定位**：足够特别、足够有设计感
- **原则**：创意优先，大胆实验，追求辨识度与记忆点
- **输入**：`design_spec` 含 `design_path: 1`，风格方向、情绪基调
- **输出**：高创意度的设计规格，可突破常规布局

### 路径 2：内容工具（设计大师参与）

- **定位**：虽然前端专家主导实现，但设计大师应积极参与，提供具体的设计建议
- **原则**：提供关于视觉层次、留白、配色、版式的具体建议，帮助前端专家提升设计质量
- **参与方式**：在头脑风暴阶段和需求确认阶段，主动提供设计建议
- **建议内容**：
  - 视觉层次：信息架构、视觉权重、内容重要性表达
  - 留白节奏：区块间距、卡片内边距、留白比例（建议具体数值，如 80px、32px 等）
  - 配色方案：主色、辅色、中性色的具体搭配建议（建议具体 hex 值）
  - 版式节奏：区块划分、间距比例、视觉流引导
  - 视觉效果：渐变、阴影、动画等效果的具体建议
- **输出**：在 `demand.md` 中记录设计建议，供前端专家参考

### 路径 3：参考+创意

- **定位**：在参考基础上做创意延伸
- **原则**：理解参考页面的结构、配色、氛围，结合用户需求做创新
- **输入**：`design_spec` 含 `design_path: 3`、**参考页面 URL 或截图描述**、需保留/创新的部分
- **输出**：在参考基础上演化的设计规格，保留有效模式、加入新创意

## 设计原则

1. **知识库驱动设计**：**必须**在设计前检索知识库，根据需求中的风格、情绪基调、行业等信息检索相关的设计知识
2. **层次清晰**：信息架构明确，视觉权重符合内容重要性，**参考知识库中的版式规律**
3. **留白得当**：避免拥挤，留白即设计，**使用知识库中的间距系统知识**
4. **配色统一**：主色、辅色、中性色协调，满足规范约束，**参考知识库中的配色体系**
5. **版式有节奏**：区块划分、间距比例、视觉流引导用户视线，**参考知识库中的布局模式**
6. **拒绝平庸**：遵循 frontend-design，选择有辨识度的风格，避免 Inter/Space Grotesk 等泛滥字体和紫色渐变，**参考知识库中的风格参考**
7. **行业适配**：遵循 ui-ux-pro-max，按产品类型选择风格、配色和排版，**参考知识库中的行业案例**

## 输入

- **路径 1、3**：来自 Orchestrator 的 `design_spec`（含 `design_path`、产品、内容、风格、规范）
- **路径 2**：来自 Orchestrator 的 `demand.md`（在头脑风暴阶段和需求确认阶段参与）
- **路径 3 必含**：参考页面 URL 或可描述参考页面的信息
- **前置条件**：`demand.md` 已存在，严格基于需求进行设计或提供设计建议

## 输出格式

产出 `design-output.md` 或等价的 YAML/JSON 设计规格，需包含：

```yaml
# 设计规格结构
meta:
  design_path: 1 | 3  # 1=纯创意 3=参考+创意
  page_name: ""
  target_audience: ""
  mood: ""  # 如：简洁、活力、专业

layout:
  sections:  # hero, features, testimonials, pricing, cta 等
    - id: hero
      type: hero
      layout: centered | split | full-bleed
      height: 100vh | auto
      components: [...]

typography:
  primary_font: ""
  heading_scale: 1.25
  line_height: 1.6

color:
  primary: "#hex"
  secondary: "#hex"
  background: "#hex"
  text: "#hex"

spacing:
  section_gap: "4rem"
  container_max: "1200px"
```

## 设计要求

### 知识库使用要求（必须执行）

**在设计前，必须执行以下步骤**：

1. **检索设计知识**：
   - 根据 `demand.md` 中的风格、情绪基调、行业等信息，检索知识库中的相关设计知识
   - 例如：需求中提到"极简风格" → 检索 `minimal-style` 节点及其依赖知识

2. **查找依赖关系**：
   - 使用 DAG 的依赖关系，查找相关的设计模式
   - 例如：选择"极简风格"时，自动查找其依赖的"留白"、"无衬线字体"、"有限配色"等知识

3. **参考行业案例**：
   - 根据产品行业，参考知识库中的行业案例
   - 例如：SaaS 产品 → 参考 SaaS 行业案例

4. **应用设计知识**：
   - 根据检索到的知识，应用相应的版式规律、配色体系、视觉语言
   - 使用知识库中的具体数值和建议（如：区块间距 80-120px、卡片内边距 32-48px 等）

### 设计输出要求

- 遵循用户给出的设计规范（色彩、字体、禁止项）
- **必须应用知识库中的设计知识**，确保设计质量和专业性
- 在约束内发挥创意，避免泛泛的「AI 风格」
- 设计规格需足够具体，便于前端专家 1:1 实现
- **路径 2 下的参与要求**：
  - 在头脑风暴阶段，**检索知识库**，主动提供关于视觉层次、留白、配色、版式的具体建议
  - 建议应包含具体的数值、比例、效果描述（**使用知识库中的标准值**，如：区块间距 80px、卡片内边距 32px、主色 #cbbfac 搭配中性灰 #fafafa 等）
  - 可以提供视觉参考或示例，帮助前端专家理解设计意图
  - 建议应记录在 `demand.md` 中，供前端专家参考

## 迭代优化流程（根据评分报告优化）

当评分 Agent 评估页面不合格，且**设计质量维度不达标**时，设计大师需要根据 `rating.md` 中的优化建议优化设计规格。

### 触发条件

- 评分 Agent 完成评分后，发现**设计质量维度得分 < 16/20（<80%）**
- 设计质量相关的优化建议中包含需要调整设计规格的问题

### 优化流程

1. **读取评分报告**：
   - 读取同目录下的 `rating.md`
   - 重点关注「三、设计质量优化建议」部分：
     - **层次清晰**：信息架构、视觉权重问题
     - **留白得当**：留白节奏、拥挤问题
     - **配色统一**：主色、辅色、中性色协调问题
     - **版式有节奏**：区块划分、间距、视觉流问题
     - **有辨识度**：避免泛化 AI 风格、提升创意度

2. **检索知识库**（必须执行）：
   - 根据评分报告中的问题，检索知识库中的相关设计知识
   - 例如：留白问题 → 检索 `generous-whitespace` 节点
   - 例如：配色问题 → 检索 `color-palette` 节点
   - 例如：版式问题 → 检索 `layout-patterns` 相关节点
   - 使用 DAG 的依赖关系，查找相关的设计模式

3. **分析设计问题**：
   - 识别设计质量不达标的具体原因
   - 确定需要优化的设计元素（布局、配色、留白、版式、风格）

4. **优化设计规格**（使用知识库中的设计知识）：
   - **层次问题**：调整信息架构，明确视觉权重，优化内容重要性表达，**参考知识库中的版式规律**
   - **留白问题**：增加或调整留白，优化间距节奏，**使用知识库中的间距系统知识**（如：区块间距 80-120px、卡片内边距 32-48px）
   - **配色问题**：调整配色方案，确保主色、辅色、中性色协调，**参考知识库中的配色体系**
   - **版式问题**：优化区块划分、间距比例、视觉流，**参考知识库中的布局模式**
   - **辨识度问题**：调整字体选择、避免泛化风格、增强创意度，**参考知识库中的风格参考**

4. **更新设计规格**：
   - 更新 `design-output.md` 或设计规格文档
   - 明确标注优化内容，便于前端专家理解改动

5. **通知前端专家**：
   - 设计规格更新后，前端专家需要根据新规格重新实现页面
   - 生成新版本页面（`landing-v2.html`、`landing-v3.html` 等）

6. **等待重新评分**：
   - 前端专家实现后，等待评分 Agent 重新评估
   - 重复此流程，直到设计质量达到合格标准

### 优化原则

- **保持设计一致性**：优化时保持整体设计风格的一致性
- **参考评分建议**：严格按照 `rating.md` 中的设计质量优化建议执行
- **使用知识库**：**必须**检索知识库，使用知识库中的设计知识和标准值，确保优化质量和专业性
- **平衡创意与规范**：在提升设计质量的同时，保持创意和辨识度
- **具体可实施**：优化后的设计规格必须足够具体，便于前端专家实现
