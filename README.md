# Landing Maker

一个基于多 Agent 协作的 Landing 页面生成系统，通过产品专家、设计大师、前端专家、SEO 专家和评分 Agent 的协作，自动生成高质量的 Landing 页面。

## 🎯 核心特性

- **多 Agent 协作**：6 个专业 Agent 各司其职，协同完成 Landing 页面生成
- **头脑风暴机制**：需求挖掘后，各专家参与头脑风暴，多角度优化需求
- **三种设计路径**：纯创意、内容工具、参考+创意，满足不同场景需求
- **自动 SEO 优化**：SEO 专家自动生成关键词和优化建议，前端专家自动实施
- **质量保证**：评分系统确保页面质量，不合格自动迭代优化
- **迭代优化**：基于评分报告自动优化，直到达到合格标准
- **项目复盘**：完成后自动复盘，持续优化各 Agent 能力

## 📋 系统架构

### Agent 体系

| Agent | Skill | 职责 |
|-------|-------|------|
| **产品专家** | `landing-orchestrator` | 通过产品文档解析和多轮对话深度挖掘需求，生成 `demand.md`；根据头脑风暴结果更新需求；**项目完成后组织项目复盘** |
| **项目经理** | `landing-project-manager` | 组织头脑风暴、协调各 Agent、管理迭代流程、任务编排、质量把关、项目复盘（规划中） |
| **设计大师** | `landing-design-master` | 路径 1、3 主导设计，产出设计规格；**路径 2 下积极参与，提供具体设计建议**；参与头脑风暴提出设计建议 |
| **SEO 专家** | `landing-seo-expert` | 基于 `demand.md` 生成关键词与 SEO 建议；参与头脑风暴提出 SEO 建议 |
| **前端专家** | `landing-frontend-expert` | 路径 2 主导实现；路径 1、3 实现设计规格；**路径 2 下主动参考设计大师建议，提升设计质量**；参与头脑风暴提出技术建议 |
| **评分 Agent** | `landing-scorer` | 按固定标准对页面打分，输出结构化评分报告 |

### 设计 Skill

- **`frontend-design`**：前端美学原则，避免泛化 AI 风格
- **`ui-ux-pro-max`**：UI/UX 规范、可访问性、Pre-Delivery Checklist

## 🛣️ 三种设计路径

| 路径 | 名称 | 主导方 | 适合场景 |
|------|------|--------|----------|
| **路径 1** | 纯创意 | 设计大师 | 追求特别、有设计感，创意优先 |
| **路径 2** | 内容工具 | 前端专家 | 内容已准备好，需要合理结构充分展示 |
| **路径 3** | 参考+创意 | 设计大师 | 有参考页面，希望在此基础上创新 |

## 🔄 工作流程

### 1. 需求挖掘阶段

**产品专家（Orchestrator）** 负责：
- 接收产品文档（可选）：解析 PRD、品牌手册等
- 多轮对话：渐进式提问，逐步明确需求
- 信息融合：将文档内容与对话信息结合
- 路径选择：引导用户选择最适合的设计路径

**输出**：`demand.md`（用户需求文档）

### 2. 头脑风暴阶段

**项目经理 Agent** 负责组织：
- 组织产品专家、SEO 专家、设计大师、前端专家参与头脑风暴
- 针对 `demand.md` 中的用户需求进行多角度分析和建议：
  - **产品专家**：从产品角度提出内容优化建议
  - **SEO 专家**：从 SEO 角度提出关键词和优化建议
  - **设计大师**：从设计角度提出视觉和交互建议
  - **前端专家**：从实现角度提出技术可行性建议
- 汇总各专家的建议，形成头脑风暴总结报告
- 将总结报告交付给产品专家

**产品专家** 负责：
- 参与头脑风暴，提出产品角度建议
- 根据头脑风暴结果更新 `demand.md`

**输出**：更新后的 `demand.md`（包含头脑风暴优化建议）

### 3. 需求确认阶段

- 产品专家向用户展示更新后的 `demand.md`
- 根据反馈修正
- **必须**待用户明确确认后再进入生成阶段

### 4. SEO 优化阶段

**SEO 专家** 负责：
- 基于 `demand.md` 生成关键词策略
- 提供 Meta 标签建议（Title、Description、Open Graph）
- 提供标题层级结构建议（H1/H2/H3）
- 提供结构化数据建议（JSON-LD Schema）
- 提供 AI-Optimized SEO 建议

**输出**：`seo-suggestions.md`（SEO 优化建议文档）

### 5. 生成阶段

根据所选路径执行：

- **路径 1（纯创意）**：
  1. 设计大师 → 产出设计规格
  2. 前端专家 → 实现设计规格（参考 SEO 建议）

- **路径 2（内容工具）**：
  1. 前端专家主导 → 直接实现（参考 SEO 建议和设计大师的设计建议）
  2. **设计大师参与**：在头脑风暴阶段提供具体的设计建议（视觉层次、留白、配色、版式等），前端专家主动参考

- **路径 3（参考+创意）**：
  1. 用户提供参考页面
  2. 设计大师 → 参考后产出设计规格
  3. 前端专家 → 实现设计规格（参考 SEO 建议）

**输出**：`landing.html`（初版页面）

### 6. 评分阶段

**评分 Agent** 负责：
- 对生成的页面进行评分
- 检查是否达到合格标准：
  - 各大项评分：每个维度得分 ≥ 该维度满分的 80%
  - 总分：≥ 85 分

**评分维度**（满分 100 分）：
- 需求符合度：25 分（≥20 分）
- UI/UX 规范：30 分（≥24 分）
- 设计质量：20 分（≥16 分）
- 技术实现：10 分（≥8 分）
- SEO 质量：15 分（≥12 分）

**输出**：`rating.md`（评分报告，包含详细得分和优化建议）

### 7. 迭代优化阶段（仅当评分不合格时）

**自动触发条件**：
- 任一维度得分 < 80%
- 或总分 < 85 分

**优化流程**：
1. 读取 `rating.md` 中的评分报告和优化建议
2. 根据未达标维度，确定需要优化的 Agent：
   - **需求符合度不达标**：产品专家检查需求理解
   - **UI/UX 规范不达标**：前端专家优化
   - **设计质量不达标**：设计大师（路径 1、3）或前端专家（路径 2）优化
   - **技术实现不达标**：前端专家优化
   - **SEO 质量不达标**：
     - SEO 专家输出质量不达标：SEO 专家优化 `seo-suggestions.md`
     - 前端专家实施质量不达标：前端专家优化页面实施
3. 相关 Agent 根据优化建议进行改进
4. 生成新版本页面（`landing-v2.html`、`landing-v3.html` 等）
5. 返回步骤 6，重新评分
6. **重复迭代，直到评分合格为止**

### 8. 项目完成检查清单（合格性检查通过后必须执行）

**产品专家**负责检查：
- [ ] 需求挖掘完成
- [ ] 头脑风暴完成
- [ ] SEO 优化完成
- [ ] 页面生成完成
- [ ] 评分完成
- [ ] 合格性检查通过
- [ ] **项目复盘完成**（必须执行）

所有检查项完成后，项目正式完成。

### 9. 项目复盘阶段（合格性检查通过后必须执行）

**产品专家（作为项目经理角色）**负责组织：
- 组织全体参与 Agent 进行项目复盘
- 参与者：产品专家、SEO 专家、设计大师、前端专家、评分 Agent

**复盘内容**：
- **需求挖掘阶段**：产品专家的表现和不足
- **头脑风暴阶段**：各专家的参与度和建议质量
- **SEO 优化阶段**：SEO 专家的输出质量和建议完整性
- **生成阶段**：设计大师和前端专家的实现质量
- **评分阶段**：评分 Agent 的评估准确性和建议有效性
- **迭代优化阶段**：各 Agent 的优化效果和响应速度

**复盘输出**：
- **问题识别**：明确各个 Agent 不足的地方
- **能力提升建议**：输出各个 Agent 能力提高的建议
- **技能优化**：根据复盘结果，优化各个 Agent 的技能（Skill 文件）
- **复盘报告**：生成项目复盘报告，记录复盘结果和优化建议

**输出**：`retrospective.md`（项目复盘报告）

**执行时机**：项目达到合格标准后立即执行，不得遗漏

## 📊 评分标准

### 合格标准

- **各大项评分**：每个维度得分 ≥ 该维度满分的 80%
- **总分要求**：≥ 85 分

### 评分维度详解

#### 1. 需求符合度（25 分）

- 设计路径匹配（5 分）
- 产品信息完整（5 分）
- 核心卖点/内容展示（8 分）
- 页面结构完整（4 分）
- 设计约束执行（3 分）

#### 2. UI/UX 规范（30 分）

- 视觉质量（7 分）：无 emoji 图标、图标统一、无布局位移、主题色使用
- 交互（7 分）：cursor-pointer、hover 反馈、过渡时长、焦点可见
- 布局（5 分）：浮动元素间距、内容不被遮挡、响应式
- 可访问性（6 分）：图片 alt、表单 label、非仅靠颜色、prefers-reduced-motion
- 反模式（5 分）：无常见反模式

#### 3. 设计质量（20 分）

- 层次清晰（4 分）
- 留白得当（4 分）
- 配色统一（4 分）
- 版式有节奏（4 分）
- 有辨识度（4 分）

#### 4. 技术实现（10 分）

- 响应式（4 分）
- 语义化（3 分）
- 无运行错误（3 分）

#### 5. SEO 质量（15 分）

- **SEO 专家输出质量（6 分）**：
  - 关键词策略完整（2 分）
  - Meta 标签建议完整（1 分）
  - 标题结构建议清晰（1 分）
  - 结构化数据建议（1 分）
  - AI-Optimized SEO 建议（1 分）

- **前端专家 SEO 实施质量（9 分）**：
  - Meta 标签实施（2 分）
  - 标题结构实施（2 分）
  - 关键词融入（1 分）
  - 图片 Alt 标签（1 分）
  - 结构化数据实施（2 分）
  - AI-Optimized 实施（1 分）

## 🔧 技术栈

- **输出格式**：纯静态 HTML/CSS/JS，无框架依赖
- **可迁移性**：单文件或 HTML + 独立 CSS/JS，便于部署到任意静态托管
- **响应式**：支持 375px、768px、1024px、1440px 等主流断点
- **SEO 优化**：JSON-LD 结构化数据、Meta 标签、语义化标签

## 🚀 快速开始

1. 在 Cursor 中描述你的需求
2. 产品专家会通过多轮对话挖掘需求
3. 确认需求后，系统自动执行生成流程

5. 如不合格，系统自动迭代优化直到合格

## 📝 注意事项

- **需求确认**：生成前必须确认 `demand.md`，确保需求准确
- **需求挖掘深度**：产品专家会主动询问视觉效果和专业性期望，深入了解用户需求
- **设计参与**：路径 2 下设计大师也会提供具体的设计建议，前端专家应主动参考
- **SEO 优化**：前端专家必须完整应用 `seo-suggestions.md` 中的所有建议
- **质量保证**：页面必须达到合格标准（各大项 ≥80%，总分 ≥85）才能交付
- **迭代优化**：评分不合格时会自动触发迭代优化流程，直到合格为止
- **项目复盘**：项目完成后必须执行项目复盘，总结经验，优化各 Agent 能力
